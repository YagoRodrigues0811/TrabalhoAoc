1 - Explique Set de Instruções, com seus elementos.

Set de Instruções é a coleção de instruções que são interpretadas pela CPU. Cada Instrução tem Quatro Elementos:
----OPCode - Especifica a operação a ser executada através de código binário;
----Source Operand Reference ("Referência de Operando Fonte") - A Operação pode envolver um ou mais operandos que são os inputs da operação;
----Result Operand Reference ("Referência ao Operando Destino") - A Operação pode produzir um resultado;
----Next Instruction Reference ("Referência da Próxima Instrução") - Indica à CPU onde fazer a busca da próxima instrução, quando a execução estiver completa.

2 - Explique a forma que uma instrução pode ser representada, de acordo com os tipos possíveis de processador.

No processamento, a instrução é interpretada em bits e nas relações com os programadores ela é representada de modo simbólico.

3 - Explique tipo de dados e qual sua relevância.

Os tipos de dados são definições de valores e operações utilizadas para se armazenar determinado dado, ou sua flexão, em memória.

4 - Explique os tipos de Operações possíveis em um computador.

São 7 os tipos de operação:
----Data Transfer ("Transferência de Dados") - Operação de Movimentação de determinado dado de uma fonte para um destino.
----Arithmetic ("Aritmética") - Operações de Manipulação de Números.
----Logical ("Lógica") - Operações bit a bit e Operações Lógicas.
----Conversion ("Conversão") - Operações de conversão de sistemas de dados.
----Input/Output I/O ("Entrada e Saída") - Operações de Entrada e Saída de dados.
----Systems Control ("Controle do Sistema") - Operações para controle do sistema operacional em determinada ação.
----Transfer of Control ("Transferência de Controle") - Operações de Alternancia do que a de ser executado.

5 - Explique as características dos processadores CISC.

Os Processadores de Arquitetura CISC têm por caracterítica principal a presença de instruções complexas, mas a execução delas se torna mais lenta.

6 - Explique as características dos processadores RISC.

Suporta menos instruções, e com isso executa com mais rapidez o conjunto de instruções que são combinadas.

7 - Explique as características dos processadores Híbridos CISC-RISC.

Nos Processadores Híbridos, o processador processa apenas instruções simples. Estas instruções internas, variam de processador para processador. Sobre estas instruções internas, temos um circuito decodificador, que converte as instruções complexas utilizadas pelos programas em várias instruções simples que podem ser entendidas pelo processador.

8 - Explique o banco de registradores dos processadores RISC e, o motivo de sua adoção.

Um banco de registradores é um componente digital composto por um conjunto de registradores, em sistema de janelas, que podem ser acessados de forma organizada. A sua adoção se dá pela maior velocidade de acesso comparado à memória cache e à memória principal.

9 - Explique Registradores de uso geral e de uso específico, dizendo, pro caso dos de uso específico, quais são e para que servem.

Os registradores de uso geral são usados para armazenar dados da execução do programa, como os valores que serão operados e o resultado das operações. Os registradores de uso específico guardam informações especiais, como por exemplo:
Program Counter: Contador de Programa;
Instruction Decoding Register: Registrador de Decodificação de Instrução;
Memory Address Register: Registrador de Endereço de Memória;
Memory Buffer Register: Registrador de Buffer de Memória.

10 - Explique, utilizando as características do ciclo de execução de instrução, Pipeline.

Um Ciclo de Execução de instruções é dividido em estágios, estes são:
Busca da Instrução
Decodificação da Instrução
Busca de Operandos
Calculo de Operandos
Execução da Instrução
Escrita de Resultados
Essas etapas podem ser executadas paralelamente, por exemplo, quando uma busca de instrução é realizada e a instrução vai para a etapa de decodificação, outra busca já pode ser realizada, e assim por diante.

11 - Explique os tipos de tratamento de saltos em Pipelines.

Multiple Streams - trata a branch em multiplos pipelines, mas multiplos pipelines tendem a causar um delay no acesso à memória
Prefetch Branch Target - instrução alvo da Branch é buscada quando reconhecida
Loop Buffer - Pequena memória que contém as instruções buscadas recentementes, se uma branch for acontecer o hardware busca a instrução no buffer.
Branch Prediction:
- Predict Never Taken - Static - Nunca Espera uma branch e sempre busca as instruções em sequência
- Predict Always Taken - Static - Sempre espera uma branch e sempre busca o alvo da branch
- Predict by Opcode - Static - Decisão baseada no opcode da instrução de branch.
- Taken/Not taken switch - Dynamic - Baseado no que já aconteceu, associa um ou mais bits a cada tipo de branch para guardar se deu ou não salto da última vez. Eficiente para loop.
- Delayed Branch - não desvia até que seja necessário. rearranja as instruções.
